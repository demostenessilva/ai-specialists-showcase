✨ Features Técnicos Enterprise

🎯 Feature Store (Tecton)

Problema Resolvido: Inconsistência entre features de treino e inferência (training-serving skew).

Solução: Offline Store (Snowflake) para treino + Online Store (Redis) para inferência <10ms.

Impacto: Modelos 25% mais precisos ao eliminar feature drift.

⚡ Latência p99 <200ms

Problema Resolvido: Recomendações lentas impactam page load time e conversão.

Solução: Feature Store Online (Redis) + SageMaker Endpoint com auto-scaling.

Impacto: Conversão aumenta 12% quando latência <250ms (Google research).

🔄 Retraining Semanal

Problema Resolvido: Modelos obsoletos com dados antigos.

Solução: Pipeline dbt + SageMaker orquestrado por Airflow (MWAA), retreina a cada 7 dias.

Impacto: Modelos sempre atualizados com tendências de compra recentes.

📐 Modelagem Kimball (Star Schema)

Problema Resolvido: Dados desorganizados e difíceis de consultar.

Solução: `fct_transactions` + `dim_customers` (SCD Type 2) + `dim_products` (hierarquia L1/L2/L3).

Impacto: Queries analíticas com p95 latency <5s.

📊 North Star: Revenue per Session

Métrica Crítica: Target $15.00 (atual $12.50).

Input Metrics: AOV, Conversion Rate (Recommended), Items per Order, CTR.

Impacto: Aumento de 20% em 12 meses → +$2.5M/ano em GMV.

🛡️ Governança (LGPD/GDPR)

📐 Modelagem Dimensional (Kimball Star Schema)

Fact Table: fct_transactions

Grain: 1 row per product in a transaction (line item)

CREATE TABLE marts.fct_transactions (
    -- Primary key
    transaction_line_id STRING PRIMARY KEY,
    
    -- Foreign keys
    transaction_date_key INTEGER NOT NULL REFERENCES marts.dim_date(date_key),
    customer_key INTEGER NOT NULL REFERENCES marts.dim_customers(customer_key),
    product_key INTEGER NOT NULL REFERENCES marts.dim_products(product_key),
    
    -- Measures
    quantity_sold INTEGER NOT NULL,
    unit_price_usd DECIMAL(10,2) NOT NULL,
    discount_amount_usd DECIMAL(10,2) DEFAULT 0,
    net_amount_usd DECIMAL(12,2) NOT NULL,
    
    -- Degenerate dimensions
    order_id STRING NOT NULL,
    
    -- Metadata
    created_at TIMESTAMP_NTZ DEFAULT CURRENT_TIMESTAMP,
    dbt_run_id STRING
);

-- =====================================================================
-- DIMENSION: dim_customers (SCD Type 2 - Historical Tracking)
-- =====================================================================
CREATE TABLE marts.dim_customers (
    customer_key INTEGER PRIMARY KEY AUTOINCREMENT,
    customer_id STRING NOT NULL,
    
    -- Attributes
    customer_name STRING,
    customer_segment STRING, -- 'High-Value', 'At-Risk', 'New'
    loyalty_tier STRING, -- 'Bronze', 'Silver', 'Gold'
    
    -- SCD Type 2 columns
    effective_date DATE NOT NULL,
    end_date DATE DEFAULT '9999-12-31',
    is_current BOOLEAN DEFAULT TRUE,
    
    -- Metadata
    created_at TIMESTAMP_NTZ DEFAULT CURRENT_TIMESTAMP
);

-- =====================================================================
-- DIMENSION: dim_products (Product Hierarchy for Recommendations)
-- =====================================================================
CREATE TABLE marts.dim_products (
    product_key INTEGER PRIMARY KEY AUTOINCREMENT,
    product_id STRING NOT NULL UNIQUE,
    product_name STRING NOT NULL,
    
    -- Product hierarchy (crucial for "similar items")
    category_l1 STRING, -- 'Electronics'
    category_l2 STRING, -- 'Cameras'
    category_l3 STRING, -- 'DSLR Lenses'
    brand_name STRING,
    
    -- Pricing
    msrp_usd DECIMAL(10,2),
    is_active BOOLEAN DEFAULT TRUE
);

AARRR Pirate Metrics (SQL Production-Ready)

-- ACQUISITION: New Customer Acquisition Cost (CAC)
SELECT 
    DATE_TRUNC('month', c.first_purchase_date) AS cohort_month,
    m.marketing_spend_usd / COUNT(DISTINCT c.customer_id) AS cac_usd
FROM marts.dim_customers c
JOIN marts.fct_marketing_spend m 
    ON DATE_TRUNC('month', c.first_purchase_date) = m.month
WHERE c.first_purchase_date IS NOT NULL
GROUP BY 1, m.marketing_spend_usd
ORDER BY 1 DESC;

-- RETENTION: Repeat Purchase Rate (Cohort-based)
WITH first_purchases AS (
    SELECT customer_id, MIN(transaction_date) AS first_purchase_date
    FROM marts.fct_transactions 
    GROUP BY 1
),
second_purchases AS (
    SELECT t.customer_id
    FROM marts.fct_transactions t
    JOIN first_purchases fp 
        ON t.customer_id = fp.customer_id 
        AND t.transaction_date > fp.first_purchase_date
)
SELECT
    DATE_TRUNC('month', fp.first_purchase_date) AS cohort_month,
    COUNT(DISTINCT fp.customer_id) AS total_customers,
    COUNT(DISTINCT sp.customer_id) AS repeat_customers,
    ROUND(100.0 * COUNT(DISTINCT sp.customer_id) 
          / COUNT(DISTINCT fp.customer_id), 2) AS repeat_rate_pct
FROM first_purchases fp
LEFT JOIN second_purchases sp ON fp.customer_id = sp.customer_id
GROUP BY 1
ORDER BY 1 DESC;

-- REVENUE: GMV (Gross Merchandise Volume) by Category
SELECT
    DATE_TRUNC('month', d.date_actual) AS month,
    p.category_l1,
    SUM(t.net_amount_usd) AS gmv_usd
FROM marts.fct_transactions t
JOIN marts.dim_products p ON t.product_key = p.product_key
JOIN marts.dim_date d ON t.transaction_date_key = d.date_key
GROUP BY 1, 2
ORDER BY 1 DESC, 3 DESC;
